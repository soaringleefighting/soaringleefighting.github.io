    2、MIPS介绍和设计理念

    2.1、介绍

    1981年，斯坦福大学教授Hennessy领导团队，设计出第一个MIPS架构的处理器。
    1984年，Hennessy教授离开斯坦福大学，创立MIPS科技公司。于1985年，设计出R2000芯片，1988年，将其改进为R3000芯片，1991年推出R4000芯片。后面陆续推出了R8000、R10000和R12000等型号。后来重点转向了嵌入式领域，发布了MIPS32 4Kc以及64位的MIPS64 20Kc处理器内核。
    MIPS的系统结构及设计理念比较先进，其指令系统经过通用处理器指令体系MIPS I、MIPS II、MIPS III、MIPS IV、MIPS V，以及嵌入式指令体系MIPS16、MIPS32到MIPS64的发展。
    MIPS32的架构是一种基于固定长度的定长编码指令集，并采用加载/存储（load/store)数据模型。除了用于访问存储器的加载/存储指令外，所有指令都对寄存器进行操作。经改进，这种架构可支持高级语言的优化执行。在路由器中，经常使用的一种MIPS架构就是MIPS32。
    MIPS处理器是八十年代中期RISC CPU设计的一大热点：在许多领域，如Sony， Nintendo的游戏机，Cisco的路由器和SGI超级计算机中使用。
    另外，中国的龙芯采用的就是MIPS指令架构。

    2.2、设计理念

    尽量利用软件办法避免流水线中的数据相关问题（比如跳转延迟，加载延迟）。
    在设计理念上，MIPS强调软硬件协同提高性能，同时简化硬件设计。

    3、MIPS体系架构


    简单总结：

    • RISC的设计思想在于简化计算机指令功能、规格化指令设计，使得各个指令分段较为均匀，且操作相对简单规整，从而提高主频。
    • 采用LOAD/STORE结构，其他指令均在寄存器之间对数据进行处理，提高处理速度。
    • 依赖软件（编译）实现优化及完成复杂功能。

    4、MIPS汇编基础

    4.1、MIPS SIMD架构(MSA)

    参考： https://www.mips.com/products/architectures/ase/simd/

    旨在加速多媒体的指令集扩展。
    MSA Features:
    • 32个16x8位、8x16位、4x32位和2x64位向量元素的向量寄存器
    • 高效的整数、定点和浮点数向量并行算术运算
    • 对绝对值操作数的操作
    • 可用的舍入和饱和选项
    • 全精度乘法和乘加
    • 整数、浮点和定点数据之间的转换
    • 完整的向量级比较和分支指令集，无条件标志
    • 向量（1D）和数组（2D）打散操作
    • 字节序无关的有类型加载和存储指令
    • 符合IEEE 754-2008浮点运算标准
    • 基本精确浮点异常信号
    • 用于有更多门/晶体管的芯片的预定义可扩展扩展
    • 结合通用编译器加速计算密集型应用程序
    • 用于消费电子应用或专用硬件未涵盖的功能的软件可编程解决方案
    • 新兴的数据挖掘、特征提取、图像和视频处理以及人机交互应用
    • 高性能科学计算

    MSA Applications:
    • Accelerates compute-intensive applications in conjunction with leveraging generic compiler support
    • Software-programmable solution for consumer electronics applications or functions not covered by dedicated hardware
    • Emerging data mining, feature extraction, image and video processing, and human-computer interaction applications
    • High-performance scientific computing

    4.2、MIPS数据类型

    • 所有MIPS指令都是32位长的。
    • 数据类型：
      • 字节：8 位 .byte
      • 半字：16位 .half
      • 字： 32位 .word
      • 双字：64位 .double
    • 单个字符用单引号，例如’b’
    • 字符串用双引号，例如“A string”

    4.3、MIPS指令格式

    一个CPU指令是一个单独的32位对齐的字，指令格式如下：

    • 立即数类型 I-type
    • 跳转类型 J-type
    • 寄存器类型 R-type

    三操作数指令形式，目标寄存器在左侧（与AT&T风格相反！）
    示例：
    entrypoint：
    addu $1, $2, $3

    4.4、MIPS指令风格

    • 注释： #
    • 常用1-99的数字标记LABEL。
    • 强烈建议使用MIPS惯用寄存器命名。但你需要包含头文件mips/regdef.h；如果选择直接使用通用目的寄存器名称，则使用$3这之类的命名方式。通用寄存器的编号从0-31。
    • 常量和字符串可以按照C风格使用。

    4.5、MIPS寄存器

    4.5.1、MIPS32寄存器

    MIPS32寄存器分为两类：通用寄存器（GPR)和特殊寄存器。

    1、 通用寄存器（GPR）
    在MIPS体系架构中有32个通用寄存器，表示为$0~$31，也可以用寄存器的名字表示。

    2、 特殊寄存器：
    MIPS32架构中定义了3个特殊寄存器。分别为PC（程序计数器）、HI(乘除结果高位寄存器）和LO(乘除结果低位寄存器）。在进行乘法运算时，HI和LO保存乘法的运算结果，其中HI存储高32位，LO存储低32位；而在进行除法运算时，HI保存余数，LO存储商。

    注：堆栈的走向是从高地址到低地址。

    4.5.2、MIPS64寄存器

    参考：http://www.ece.lsu.edu/ee4720/mips64v1.pdf 2.8.4 CPU Registers
    MIPS64的CPU寄存器包括：

    • 32个64位的通用寄存器(GPRs)：r0~r31
      其中，r0固定为0，r31用作JAL/BLTZAL等的目的寄存器，存放返回地址。
    • 一对特殊寄存器：HI, LO
    • 程序计数器：PC

    一个MIPS64位处理器总是产生一个64位结果。

    MIPS64的FPU寄存器包括：

    • 32个64位浮点寄存器(FPRs): f0~f31
    • 5个FPU控制寄存器: FCR0/FCR25/FCR26/FCR28/FCSR

    4.5.3、MIPS32和MIPS64寄存器区别

    MIPS64的FPU寄存器包括：

    • 32个64位浮点寄存器(FPRs): f0~f31
    • 5个FPU控制寄存器: FCR0/FCR25/FCR26/FCR28/FCSR

    3.5.3 MIPS32和MIPS64寄存器区别
    32位的为w: word
    64为的为d:double
    比如：
    运算类指令： add—> dadd, sub/mul/div—> dsub/dmul/ddiv
    访存类指令：lw—>ld, sw—> sd

    4.6、MIPS调用约定

    参考：https://zh.wikipedia.org/wiki/MIPS%E6%9E%B6%E6%A7%8B
    MIPS有几个调用约定，尤其是在32位平台上。
    O32 ABI是最常用的ABI，因为它是MIPS的原始System V ABI[33][34]。它严格基于堆栈，只有四个寄存器$a0-$a3可用于传递参数。堆栈上的空间是保留的，以防被调用者需要保存其参数，但调用者不会将寄存器存储在那里。返回值存储在寄存器$v0中；第二个返回值可以存储在$v1中。ABI形成于1990年，最后一次更新是在1994年。这种肉眼可见的缓慢，以及只有16个寄存器的老式浮点模型，促进了许多其他调用约定的扩散。它仅针对32位MIPS定义，但GCC创建了一个名为O64的64位变体。
    对64位，最常用的是SGI的N64 ABI。最重要的改进是现有八个寄存器可用于参数传递；它还将浮点寄存器的数量增加到32个。还有一个称为N32的ILP32版本，它使用32位指针来处理较小的代码，类似于x32 ABI。两者都在CPU的64位模式下运行[35]。N32和N64 ABI将前八个参数传递给寄存器$a0-$a7中的函数；后续参数在堆栈上传递。返回值（或指向它的指针）存储在寄存器$v0中；第二个返回值可以存储在$v1中。在N32和N64 ABI中，所有寄存器都被认为是64位宽。
    对于O32及N32/N64，返回地址存储在$ra寄存器中。这是使用JAL（跳转和链接）或JALR（跳转和链接寄存器）指令自动设置的。（非叶子）MIPS子例程的函数序言将返回地址（在$ra中）推入堆栈。
    在O32和N32/N64上，堆栈向下增长，但N32/N64 ABI需要所有堆栈条目64位对齐。帧指针（$30）是可选的，实际上很少使用，除非需要在运行时确定函数中的堆栈分配，例如调用了alloca()。
    对于N32和N64，返回地址通常存储在栈指针之前的8个字节，尽管这可能是可选的。
    对于N32和N64 ABI，函数必须保留$S0-$s7寄存器、全局指针（$gp或$28）、栈指针（$sp或$29）和帧指针（$30）。O32 ABI相同，只是$gp寄存器需要由调用函数保存而不是被调用函数。
    对于多线程代码，线程本地存储指针通常存储在特殊硬件寄存器$29中，并通过使用mfhw（从硬件移动）指令访问。已知至少有一个供应商将此信息存储在$k0寄存器中，该寄存器通常保留供内核使用，但这不是标准的。
    $k0和$k1寄存器（$26-$27）保留供内核使用，不应由应用程序使用，因为内核可以随时由于中断、上下文切换或其他事件而更改这些寄存器。

    4.7.1、寻址方式

    MIPS架构的寻址模式非常简单，就是寄存器+偏移量的方式，偏移量的范围是−32768~32767（也就是16位的立即数）。如果编程人员想要使用其它寻址方式，汇编器将会使用寄存器+偏移量的方式进行组合实现。

    • 寄存器寻址：add $t0, $t1, $t2
    • 立即数寻址：addi $sp, $sp, 4
    • 基址或偏移寻址： Lw $t0, 8($s0)
    • PC相对寻址：bne $s0, $s1, Exit
    • 伪直接寻址：？

    4.7.2、MIPS指令特点

    • MIPS固定4字节指令长度；
    • 内存中的数据访问（load/store）必须严格对其（至少4字节对齐）；
    • 跳转指令只有26位目标地址，加上2位对齐位，可寻址28位的空间，即256MB；
    • 条件分支指令只有16位跳转地址，加上2位对齐位，可寻址18位的空间，即256KB；
    • 流水线效应。MIPS采用了高度的流水线，其中最重要的就是分支延迟效应。在分支跳转语句后面那条语句叫分支延迟槽。实际上，在程序执行到分支语句时，当他刚把要跳转的地址填充好（填充到代码计数器里），还没有完成本条指令时，分支语句后面的那个指令就已经执行了，其原因就是流水线效应——几条指令同时执行，只是处于不同的阶段，mips不像其它架构那样存在流水线阻塞。所以分支跳转语句的下一条指令通常都是空指令nop或一些其他有用的语句。
    • 缓存刷新机制：MIPS CPUs有两个独立的cache:指令cache和数据cache。 指令和数据分别在两个不同的缓存中。当缓存满了，会触发flush, 将数据写回到主内存。
    • MIPS64开头以“d”表示，无符号数以“u”结尾，立即数通常以“i”结尾，字节操作以“b”结尾，双字操作以“d”结尾，字操作以“w”结尾。

    4.7.3、存取指令

    LB/LH/LW/LD/L.S/L.D/LBU/LHU/LWU/LI/LUI
    SB/SH/SW/SD/S.S/S.D

    LB：从存储器中读取一个字节的数据到寄存器
    LI(Load Immediate)：用于将一个最大16位立即数存入一个寄存器。Eg. Li $Rd, imm
    LUI(Load Upper halfword Immediate): 读取一个16位立即数放入寄存器的高16位，低16位补0。Lui $a1, 0x42

    LL/SC指令：
    LL(Load Linked) : 链接加载
    SC(Store Conditional)：条件存储
